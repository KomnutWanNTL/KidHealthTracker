แปลงเนื้อหาวิศวกรรมนี้สำหรับผู้บริหาร:

$ARGUMENTS

**ถามก่อน:** channel ไหน? (JIRA / Slack / standup / email / meeting)

**หลักการ:**
- เก็บ: product names, JIRA keys, PR numbers, customer/workload identifiers
- ตัด: function names, file paths, commit SHAs, code jargon
- แปล mechanism เป็น plain English cause-and-effect
- active voice, short paragraphs, no hedging

**ต่อ channel:**
- JIRA: structured (Status/Impact/What broke/Owner/Next steps)
- Slack: bold TL;DR + 2-4 bullets ~80 คำ
- Standup: 1-3 lines, front-load verb
- Email: subject = TL;DR, flowing paragraphs
- Meeting: bullet points, max 1 clause/bullet
